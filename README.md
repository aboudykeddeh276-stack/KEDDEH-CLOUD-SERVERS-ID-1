# KEX HyperDrive Dashboard UI

A highly engineered, modular dashboard skeleton for a front-facing KEX/BRAINK runtime control plane.

## Purpose

```text
Knowledge Sheets
→ Front-Facing Address Space
→ KEX Query Engine / Router
→ Folder Substrate
→ SSD / Nested KSSD
→ Hyper Computers
→ GPU / MUX Screen
→ Host Chat
→ Ledger / Proof Capsule
```

## Run

```bash
npm install
npm run dev
```

## Populate

Edit:

```text
src/data/placeholders.ts
```

Then wire real data through:

```text
src/lib/adapters/
```

## Claim boundary

This is a frontend skeleton. It does not create real hardware, execute arbitrary commands, or mutate folders until you connect a real watcher/router service.
