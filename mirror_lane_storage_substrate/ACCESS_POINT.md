# Access Point

## Role

This file is the entry point below README for the Markdown substrate.

It routes the reader or tooling into the cell-native storage substrate and then into the FAT-to-KEX requirements engine.

## Corrected claim

The substrate is not only a disk-requirements map.

The substrate is written into cells, Markdown files, and linked stacks.

Those cells change state. The files carry the cells. The Markdown stack carries readable and parseable structure. The drive remains the host medium, but the effective substrate is the software cell layer.

## Boundary

This does not replace physical SSD or HDD hardware at the electronics level.

It replaces the logical storage surface for this stack by using cell-state, file-state, and Markdown-state as the working substrate.

The disk remains physical hardware. The KEX substrate is a software cell layer that names, routes, pairs, checks, and indexes state.

## Engine rule

When FAT adaptation occurs, the engine maps disk concepts into cell-native software requirements.

Physical storage remains the host.

KEX adds a cell-native mirror-lane substrate above the base storage system.
